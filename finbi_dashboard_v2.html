Como editar este dashboard
Este arquivo é um HTML puro — você pode editá-lo com qualquer editor de texto. Veja abaixo os pontos mais comuns para personalizar.
1. Abrir o arquivo para edição
Clique com o botão direito no arquivo finbi_dashboard.html → Abrir com → Bloco de Notas (Windows) ou TextEdit (Mac).
Para edição avançada, use o VS Code (gratuito em code.visualstudio.com).
2. Mudar as cores do painel
Procure por :root { no início do arquivo (linha ~10).
Altere os valores hexadecimais das variáveis:
--accent: #3b82f6 → cor principal (azul)
--green: #10b981 → cor de ganhos
--red: #f43f5e → cor de perdas
--bg: #080c14 → fundo escuro
3. Mudar o nome do sistema
Procure por FIN<span>BI</span> para alterar o logo.
Procure por <title>FinBI para mudar o título da aba do navegador.
4. Editar dados de demonstração
Procure por function getDefaultDB().
Dentro de transacoes: [ estão os dados iniciais de exemplo.
Edite ou apague as linhas para ajustar aos seus ativos. Use o formato:
{id:1, data:'2024-01-15', ticker:'PETR4', tipo:'COMPRA', classe:'ACAO_BR', qty:100, preco:38.50, taxa:4.90, corretora:'XP'}
5. Atualizar preços manualmente
Procure por precos: {.
Altere os valores para os preços atuais dos seus ativos:
PETR4: 36.80, VALE3: 65.40, BTC: 320000
Após salvar e reabrir o arquivo, os cálculos serão atualizados.
6. Salvar e recarregar
Após editar: salve o arquivo (Ctrl+S) e atualize o navegador (F5).
Atenção: se você já usou o painel e tem dados no navegador, os dados de demonstração do código só aparecem ao limpar os dados em Configurações → Limpar todos os dados.

Como funciona cada tipo de evento
📈 Desdobramento (Split)
Multiplica a quantidade e divide o PM pelo fator.
Ex: 1:10 → qty ×10, PM ÷10
📉 Grupamento (Inplit)
Divide a quantidade e multiplica o PM pelo fator.
Ex: 10:1 → qty ÷10, PM ×10
🎁 Bonificação
Acrescenta ações gratuitas ao saldo. O custo total não muda, o PM é recalculado.
Ex: 10% → qty ×1.1, PM ×(1÷1.1)
📋 Subscrição
Registra a compra de novas ações pelo preço de emissão. Tratada como nova COMPRA.
🔄 Conversão de Classe
Troca o ticker (ex: PN→ON) ou o código do ativo. Mantém qty, PM e custo.
🔀 Fusão / OPA / M&A
Encerra a posição no ticker antigo e cria (ou não) posição no novo, pela relação de troca definida.
